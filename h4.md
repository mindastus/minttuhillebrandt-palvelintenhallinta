## h4 Pizza Fantasia

## x) 4.12.1 Size and Complexity of Some DSLs
- *Salt DSL:* 510 state-funktiota, dokumentaatio yli 20 000 riviä / 75 000 sanaa – isompi kuin väitöskirja.
- *Puppet:* 113 funktiota, mutta omat kontrollirakenteet (esim. virtual resources).
- *Yhteenveto:* Suuret CM‑DSL:t ovat raskaita ja vaikeita hallita.
- *Oma huomio:* Tarvitaanko oikeasti näin isoja DSL:iä, jos käyttäjät käyttävät vain murto-osaa?

## x) 4.12.2 Use of DSL Functions in Case Configuration
- Todellisissa Puppet‑konfiguraatioissa 80–90 % käytöstä on samoja perusjuttuja:
`file, package, service, exec, if/case.`
- *USGCB:* samat perusfunktiot + augeas.
- Suuri osa DSL:n ominaisuuksista jää käyttämättä.
- *Oma kysymys:* Jos käytetään vain muutamaa perusfunktiota, pitäisikö työkalujen olla paljon yksinkertaisempia?

## x) 4.12.3.1 Dependencies Between Main Functions
- Kaikki rakentuu kahden perusasian päälle: `file` ja `exec`.
- Näistä muodostuvat: `package`, `service`, `user`, `group`.
- Idempotenssi = tarkista → tee muutos vain jos tarvitaan.
- Korkeamman tason resurssit (esim. cron) voidaan rakentaa näiden päälle.
- *Oma huomio*: Koko CM-järjestelmä voidaan pohjimmiltaan rakentaa vain kahdesta peruspalikasta.

Asensin MariaDB-palvelimen käsin. Suoritin komennot `apt-get install mariadb-server`, käynnistin ja tarkistin tilan komennolla `systemctl status mariadb`, joka näytti tilaksi active (running). Testasin toiminnan kirjautumalla sisään komennolla `sudo mariadb` ja ajamalla kyselyn `SELECT VERSION();`.
