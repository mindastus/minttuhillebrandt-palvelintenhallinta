## x) Passwordless Sudo with Ansible

- Ansible-rooli automatisoi passwordless sudo -asetuksen: luo käyttäjän, ryhmän (sudoless) ja lisää sudoers-säännön.
- `become: true` mahdollistaa sudo-oikeuksien käytön Ansible-playbookissa.
- Rooli tekee kolme pääasiaa: luo ryhmän, luo käyttäjän ja lisää ryhmiin sekä lisää SSH-avain + sudoers-tiedosto

## x) Ansible-moduulien tiivistelmät

**copy**
- Kopioi tiedostoja ohjauskoneelta kohdekoneelle.
- Tukee myös `content`-parametria, jolla voi kirjoittaa sisältöä suoraan ilman erillistä tiedostoa.

*Keskeiset optiot*
- content | suoraan moduulissa annettava tiedoston sisältö.
- src | lähdetiedosto ohjauskoneella.
- dest | kohdepolku etäkoneella.
- owner | group, mode – omistajuus ja oikeudet.

*Esimerkki*
```
- name: Copy a file
  copy:
    src: /tmp/foo
    dest: /etc/foo
```

## apt
- Hallitsee Debian/Ubuntu-pohjaisten järjestelmien paketteja.
- Voi asentaa, poistaa ja päivittää paketteja.

*Keskeiset optiot*
- name | paketin nimi.
- state | `present`, `absent`, `latest`.
- update_cache | päivittää pakettivarastot.

*Esimerkki*
```
- name: Install latest version of nginx
  apt:
    name: nginx
    state: latest
```


## file
- Luo, poistaa tai muokkaa tiedostoja ja hakemistoja.
- Hyvä oikeuksien ja polkujen hallintaan.

*Keskeiset optiot*
- path | kohteen polku.
- state | `file`, `directory`, `absent`, `link`.
- src | linkin lähde.
- recurse | muuttaa hakemiston sisällön oikeuksia.
- owner | group, mode – omistajuus ja oikeudet.

*Esimerkki*
```
- name: Ensure directory exists
  file:
    path: /etc/myapp
    state: directory
```

## user
- Luo, poistaa ja hallitsee käyttäjätilejä.
- Voi määrittää kotihakemiston, ryhmät ja shellin.

*Keskeiset optiot*
- name | käyttäjän nimi.
- create_home | luodaanko kotihakemisto.
- comment | kuvaus.
- groups | ryhmät, joihin käyttäjä liitetään.
- shell | käyttäjän shell.
- state | `present` tai `absent`.
- system | luodaanko system-käyttäjä.

*Esimerkki*
```
- name: Add user
  user:
    name: deploy
    create_home: yes
```

## authorized_key
- Lisää tai poistaa SSH-julkisia avaimia käyttäjän `~/.ssh/authorized_keys` -tiedostosta.

*Keskeiset optiot*
- user | käyttäjä, jolle avain lisätään.
- key | julkisen avaimen sisältö.

*Esimerkki*
```
- name: Add SSH key
  authorized_key:
    user: ubuntu
    key: "{{ lookup('file', 'id_rsa.pub') }}"
```

## a) Sudoless. Tee ansiblea varten tunnus, jolla voi käyttää sudoa ilman salasanaa. Sekä ssh-kirjautuminen että sudon käyttö tulee olla ansbilea varten automatisoitu.

<img width="1237" height="658" alt="Kuva virtuaalikoneen emulaattorista." src="https://github.com/user-attachments/assets/158f07c2-127a-455e-95f8-3dd940a05574" />
<img width="453" height="94" alt="Kuva virtuaalikoneen emulaattorista." src="https://github.com/user-attachments/assets/aba58110-560d-4efd-a287-d1da6e395c42" />

Tein passwordless sudo -asetuksen testauksen. Loin ryhmän `sudoless` ja lisäsin käyttäjän (*anster*) eri ryhmiin (`sudo`, `adm`, `sudoless`). Tämä antaa käyttäjälle mahdollisuuden käyttää sudoa.
Kirjauduin sisään käyttäjänä (`ssh anster@localhost`). Tämä on tärkeää, koska ryhmämuutokset tulevat voimaan uudessa sessiossa. Testasin sudoa komennolla `sudo echo moi` ja se toimi tulosti "moi".
Käytin `sudo -k` tyhjentämään välimuistin ja testasin uudelleen. Varmistin, ettei toiminta johdu aiemmin tallennetusta kirjautumisesta.

## d) File. Kirjoita orjalle useamman rivin mittainen tiedosto Ansiblella. Määrittele sen omistaja, omistava ryhmä ja oikeudet. Käytä oikeuksille oktaalinumeroa, esim. "0600". Kerro, mitä oikeudet ovat symbolisessa muodossa, esim. "-rwxr--r--". Selitä, mitä kukin käyttäjä saa tehdä tuolle tiedostolle.

<img width="1234" height="600" alt="Kuva virtuaalikoneen emulaattorista." src="https://github.com/user-attachments/assets/f6a8bf5c-4509-474f-8e33-1d01e09a5718" />

Tiedosto luotiin Ansiblella ja siihen kirjoitettiin useita rivejä sisältöä. Samalla määritellään omistaja, ryhmä ja oikeudet, esimerkiksi "0640", joka vastaa muotoa -rw-r-----. Tällöin omistaja saa lukea ja muokata tiedostoa, ryhmä saa vain lukea ja muut eivät pääse siihen lainkaan.
